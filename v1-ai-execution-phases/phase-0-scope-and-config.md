# Phase 0：范围与配置骨架

这份文件用于 AI 单次执行 `Phase 0`。

## 本次目标

固定 V1 的业务边界、运行模式和基础配置骨架，让后续实现不再朝“通用平台”扩散。

## 前置依赖

- 无

## 必读上下文

- `./00-master-plan.md`
- `../ai-quant-trading-architecture.md`

## 允许修改范围

- `configs/`
- `src/config/`
- `src/shared/` 中少量与配置直接相关的代码
- 范围说明文档
- `README.md` 中与启动配置直接相关的少量内容

## 本次必须完成的任务

- 固定交易所、市场类型、交易对、周期、运行模式
- 明确 `paper trading` 是 V1 唯一执行模式
- 建立基础配置结构，例如 `base`、`dev`
- 建立环境变量读取或等价设置方式
- 建立最小日志配置
- 写清楚默认边界：单交易所、单账户、单策略、Bar 驱动

## 本次不要做

- 不接交易所 API
- 不建立数据库表
- 不定义完整策略实现
- 不实现 trader 主循环
- 不引入 Redis、消息系统、监控平台

## 完成标准

- 项目中已经存在清晰的配置骨架
- V1 范围被固定在配置或文档中
- 后续 AI 不需要再猜“第一版到底做多大”

## 最低验证要求

- 能成功读取配置
- 配置项有默认值或明确缺省策略
- 至少有 1 个简单测试覆盖配置加载

## 本次交付时必须汇报

- 新增或修改了哪些配置文件
- V1 范围是如何被固定下来的
- 还有哪些决策仍需人工确认
