# SignalArk 回测策略优化待办

生成时间：2026-04-10

## 评估范围

- 仅聚焦当前仓库已经落地的回测与策略实现，不扩展到真实券商、多账户、多策略编排或 AI/ML 训练平台。
- 重点记录“当前回测策略还有哪些高价值优化空间”，并按优先级整理成后续 AI 易于执行的待办。
- 本清单区分“策略信号优化”和“回测研究能力优化”，避免把两类问题混在一起。

## 当前判断

- 当前默认主策略仍是 `baseline_momentum_v1`，核心规则非常简单：只比较 `close` 与 `previous_close`，超过阈值就持有固定股数，否则直接清仓。
- 当前 research/backtest 能力已经能复用生产链路中的策略、下单、成交和账本语义，但定位仍然是“最小一致性验证”，不是完整研究平台。
- 因此当前优化空间不是单点问题，而是同时存在于“信号质量”“仓位管理”“回测真实性”“绩效评估深度”四个层面。
- 如果只能先做少数事项，优先顺序应当是：先增强基线策略可解释收益逻辑，再扩展研究指标与样本验证，最后再做更重的参数搜索或 AI 策略实验。

## P0：优先提升基线策略有效性

- [x] done：给 `baseline_momentum_v1` 增加独立的离场阈值，而不是只要不满足入场条件就立即清仓。
  目标：减少策略在阈值附近来回翻仓的噪音交易。
  完成标准：策略配置中至少同时存在 `entry_threshold_pct` 和 `exit_threshold_pct`，并补齐对应单元测试与回测审计字段。

- [x] done：给基线策略增加多 bar 趋势确认，而不是只看单根 bar 相对昨收的变化。
  目标：避免单根 K 线偶发波动直接触发持仓切换。
  完成标准：策略至少支持一种简单确认机制，例如最近 `N` 根收盘趋势、短周期累计涨跌幅或均线方向，并在配置中显式可调。

- [x] done：把固定 `400` 股目标仓位改为可配置的分级仓位或动态仓位。
  目标：让仓位大小对波动、资金规模或信号强弱有响应，而不是所有市场状态都使用同一仓位。
  完成标准：至少支持“弱信号小仓位、强信号大仓位”或“按资金占比计算目标仓位”的一种方案，并在回测元数据中保留参数。

- [x] done：给基线策略补最小风险退出机制。
  目标：即使信号没有立刻翻空，也能对单边不利走势做保护。
  完成标准：至少支持一种简单的止损、回撤退出或时间止盈/止损规则，并补充对应回测案例。

## P1：增强回测研究结论的可信度

- [x] done：扩展回测绩效指标，不再只停留在收益、回撤、胜率和换手。
  目标：让研究结论不只看“赚了多少”，还要看“赚得是否稳”。
  完成标准：至少增加 Sharpe 或等价波动调整收益指标、收益回撤比、平均持有周期、单笔盈亏分布中的一部分。

- [x] done：把 research 默认样本从“快速预览”升级为“可区分预览和评估”的两层模式。
  目标：避免短样本结果被误当成策略已经稳定有效。
  完成标准：保留小样本预览能力，同时新增更长历史区间的评估入口，并在返回结果中显式标记样本用途和 bar 数量。

- [x] done：为基线策略增加时间分段验证。
  目标：判断策略是否只在某一小段行情下偶然有效。
  完成标准：至少支持按不同时间窗口导出多段回测结果，能够比较不同阶段收益、回撤和交易次数差异。

- [x] done：补充多市场状态下的策略回测样例。
  目标：不要只在少量上涨或震荡片段里验证策略。
  完成标准：测试或样例数据至少覆盖上涨、下跌、震荡三类典型区间，并能看到策略在不同区间的行为差异。

## P1：增强回测成交假设的真实性

- [x] done：让滑点模型不只支持固定 bps。
  目标：降低“成交结果过于理想化”导致的研究偏差。
  完成标准：至少支持基于方向、波动或 bar 振幅的分层滑点方案之一，同时保留当前固定 bps 作为基线对照。

- [x] done：评估是否需要引入部分成交或成交失败情景。
  目标：让回测更接近真实交易中的执行不确定性。
  完成标准：至少形成一版明确结论，决定是继续保持单次整笔成交，还是补一个最小的部分成交场景模型。

- [x] done：明确 current backtest 与 paper/live 在执行层的剩余差异。
  目标：优化策略时知道哪些结果来自 alpha，哪些结果只是模拟假设。
  完成标准：在文档或 manifest 说明中列出当前成交假设、滑点假设和未覆盖的真实执行约束。

## P2：提升研究迭代效率

- [ ] 增加参数扫描能力，用于比较不同阈值、lookback 和仓位参数组合。
  目标：减少手工反复改 YAML 再单次运行的低效方式。
  完成标准：至少支持对一组参数组合批量运行回测，并输出可比较的摘要结果。

- [ ] 增加 walk-forward 或时间序列滚动验证能力。
  目标：避免在同一段样本上反复调参后得到失真的最优结果。
  完成标准：至少形成最小版本的滚动训练/验证切分或滚动评估脚本。

- [ ] 为 AI 策略实验区增加和基线策略的标准化对照。
  目标：让 `ai_bar_judge_v1` 不只是能跑，而是能与基线策略在同样样本和指标下横向比较。
  完成标准：同一份 research 输出中能明确比较 baseline 与 AI 方案的收益、回撤、交易次数和关键决策差异。

## 暂不纳入当前待办

- 真实券商执行优化
- 多账户回测
- 多策略组合优化
- 独立训练平台
- 全量实验管理系统

## 推荐执行顺序

1. 先改 `baseline_momentum_v1` 的出入场和仓位逻辑。
2. 再补 research 指标与长样本评估入口。
3. 然后再做成交模型增强与参数扫描。
4. 最后把 AI 策略实验接入统一对照框架。
